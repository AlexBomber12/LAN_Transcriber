The current problem is not color, spacing, or rounded corners. The current problem is composition and role confusion. The screen still feels like stacked mini-interfaces instead of 1 operator workflow. The Control Center must stop introducing itself and must stop explaining itself. It must immediately act as the daily operator workspace for the loop Upload -> queue triage -> choose recording -> make immediate decisions -> open deep review only when needed. The Full Page Recording Inspector must remain the deep-review surface for transcript, diagnostics, detailed speaker work, summary validation, and export. The bottom system bar must remain the only place where DGX Spark and GPU runtime truth is shown.

Enforce these screen roles strictly. Control Center is the daily operator workspace only. Its body may contain only 3 functional zones: compact upload or intake, recordings worklist as the operational inbox, and compact selected-recording decision pane on the right. It must not contain hero sections, explanatory paragraphs, focus-story cards, duplicate summary layers, dashboard storytelling, admin snapshots, instructional blocks that re-explain the screen, or full-page detail content embedded inside it. Full Page Recording Inspector is the deep-review destination for 1 recording. Keep transcript, diagnostics, richer export, and any content that requires wide reading or long comparison on the full page. System Bar is the persistent runtime truth layer at the bottom only and must continue to carry DGX online or offline, GPU state, inference mode, active jobs, and model information if already available.

Delete aggressively from the Control Center anything that behaves like a narrative layer instead of a work layer. Remove the workspace hero or header layer if it still exists in any form. Remove explanatory paragraphs, focus cards, duplicate status summaries, dashboard-like mini-story sections, speaker instructional intro cards that repeat every time, and any “what this page is” or “how to use this page” prose. Remove any mini-dashboard layer above the actual work area. Remove any full-page-only content that leaked into compact mode. Prefer deleting UI elements over adding new ones.

Keep only what directly supports the daily workflow. Keep the top navigation. Keep the compact upload entry. Keep a compact filter row. Keep the recordings worklist. Keep the compact selected-recording pane. Keep the bottom system bar and do not move DGX or GPU status anywhere else. Keep the current Stitch-aligned visual tokens and general aesthetic. Keep existing backend behavior wherever possible.

Refactor the worklist into a strict operational inbox. The worklist must support triage, not storytelling. Keep only fields that directly support triage: Name, Status, Progress, Captured, Duration, Source, Action. Do not bring Confidence back as a primary worklist column. Reduce decorative filter and summary clutter above the table. If status pills or secondary counts duplicate what the filters or table already communicate, remove or collapse them. The selected row state must be visually obvious without adding more explanation.

Refactor the compact inspector into a true decision pane. It must be decision-first, not reading-first. In compact mode, Overview must show only current stage, blocker, next required action, and minimal key metadata. Speakers must be snippet-first, compact, and action-oriented. Summary must be short and operational. Export must remain minimal. Anything that needs deep reading, deep comparison, long transcript reading, or wide tables belongs to the full-page inspector only. Do not let the compact inspector become a mini analytical dashboard or a narrow copy of the full page.

Refactor Speakers in compact mode so the user sees snippets immediately and can act immediately. Show snippet previews, current state, and the 4 existing distinct actions: Confirm match, Keep unknown, Local label only, Add trusted sample. Preserve the current semantic separation between identity mapping and trusted-sample creation. Do not reintroduce a giant spreadsheet-like table in compact mode. Remove repetitive instructional copy if the interaction is already clear from the UI itself. The compact speakers view should read as “I hear this person -> I see the best snippet evidence -> I choose the safest action”.

Leave full-page-only content on the Full Page Recording Inspector. Keep Transcript as a full-width working tab with scroll there. Keep Summary separate there. Keep Diagnostics separate there. Keep richer Export there. Keep any advanced speaker operations there if they require more width or density. Do not pull those concerns back into the Control Center body. Do not create page-inside-page composition again.
