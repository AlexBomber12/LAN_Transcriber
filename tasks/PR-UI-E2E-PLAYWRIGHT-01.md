    PR-UI-E2E-PLAYWRIGHT-01
    =======================

    Prompt (copy as-is into Codex)
    ------------------------------

    ```text
    You are Codex Agent working on the LAN-Transcriber repository.

Goal
Add real browser-level UI smoke tests using Playwright for the most important user path:
Upload -> Recordings list -> Recording detail -> Export ZIP download.
Run these tests in CI as a separate job so they do not affect Python coverage gating.

Context
Current UI tests use FastAPI TestClient and validate HTML responses, which does not cover actual browser behavior, HTMX interactions, or static assets.
A lightweight Playwright smoke suite catches UI regressions that server-side tests miss.

Non-goals
- Do not run heavy ML processing in UI tests.
- Do not require HF_TOKEN or internet.
- Keep the suite minimal and stable (smoke tests, not exhaustive UI coverage).

Implementation requirements

1) Add Playwright tests
- Create a folder tests_playwright/.
- Add tests_playwright/test_ui_smoke_playwright.py using Playwright Python.
- The test must:
  - start the app (uvicorn lan_app.api:app) as a subprocess on a free port
  - set env vars so the app uses a temp data root + sqlite DB under a temp directory
  - wait for /healthz/app to return 200
  - open /upload in Chromium
  - upload a small synthetic WAV (generated by the test)
  - navigate to /recordings and verify the uploaded recording appears
  - open the recording detail page
  - click the Export ZIP link/button and verify a download is created
  - optionally open the ZIP and assert required files exist (manifest.json, onenote.md or your expected names)

2) CI job
- Update .github/workflows/ci.yml to add a new job: playwright-smoke
- Steps:
  - checkout
  - setup python 3.12
  - pip install playwright pytest requests
  - python -m playwright install --with-deps chromium
  - run pytest -q tests_playwright/test_ui_smoke_playwright.py
- Ensure this job runs on pull_request and push.

3) Stability
- The test should use explicit waits for UI elements and network idle (avoid sleep-based flakiness).
- Ensure the uvicorn subprocess is terminated in a finally block.
- Use a hard timeout for the test (pytest-timeout or manual wait loop).

Deliverables
- Playwright smoke test file(s).
- New CI job that runs Playwright smoke tests.

Success criteria
- CI runs Playwright smoke tests and they pass reliably.
- The tests fail if upload flow, recordings list, export ZIP, or static assets break.
    ```
